# داستان‌های کاربر - سامانه ثبت‌نام دانشگاه

## ۱. انتخاب واحد (نقش: دانشجو)
**به عنوان یک** دانشجو،  
**می‌خواهم** دروس مورد نظر خود را از لیست ترم جدید انتخاب کنم،  
**تا بتوانم** برنامه تحصیلی نیم‌سال جدید خود را تنظیم و نهایی کنم.

### بررسی INVEST
| معیار | وضعیت | توضیحات |
| :--- | :---: | :--- |
| **I**ndependent | ✅ | این بخش وابستگی مستقیمی به سایر بخش ها (مانند سیستم پرداخت شهریه) ندارد و می‌تواند به صورت مجزا توسعه یابد. |
| **N**egotiable | ✅ | جزئیات پیاده‌سازی مانند ظاهر رابط کاربری یا تعیین سقف انتخاب واحد، در طول توسعه قابل بررسی و تغییر است. |
| **V**aluable | ✅ | این بخش نیاز اصلی دانشجو را برطرف می‌کند و بدون آن، سامانه کارایی اصلی خود را برای دانشجویان نخواهد داشت. |
| **E**stimable | ✅ | پیچیدگی این عملیات (افزودن یک رکورد به لیست دروس دانشجو) مشخص است، بنابراین زمان و تلاش مورد نیاز برای برنامه‌نویسی آن به راحتی قابل تخمین است. |
| **S**mall | ✅ | دامنه این داستان محدود به «اخذ درس» است؛ عملیات دیگری مانند «حذف درس» در داستان‌های مجزایی بررسی می‌شوند. |
| **T**estable | ✅ | با شبیه‌سازی کلیک روی دکمه ثبت و بررسی اضافه شدن درس به لیست دانشجویان در پایگاه داده، به سادگی قابل تست است. |

## ۲. دیدن لیست کلاس (نقش: استاد)
**به عنوان یک** استاد،  
**می‌خواهم** لیست دانشجویانی که در کلاسم ثبت‌نام کرده‌اند را ببینم،  
**تا بتوانم** برای مدیریت کلاس، حضور و غیاب و ارزیابی دانشجویان برنامه‌ریزی کنم.

### بررسی INVEST
| معیار | وضعیت | توضیحات |
| :--- | :---: | :--- |
| **I**ndependent | ✅ | این ویژگی صرفاً یک صفحه نمایش اطلاعات است و با پیچیدگی‌های فرآیند ثبت‌نام تداخلی ندارد. |
| **N**egotiable | ✅ | جزئیات نمایش داده‌ها انعطاف‌پذیر است؛ به عنوان مثال، می‌توانیم بعداً تصمیم بگیریم که عکس دانشجویان نیز در کنار نامشان نمایش داده شود. |
| **V**aluable | ✅ | این قابلیت به استاد کمک می‌کند تا پیش از شروع کلاس، شناخت دقیقی از دانشجویان خود داشته باشد. |
| **E**stimable | ✅ | این عملیات شامل یک جستجوی ساده در پایگاه داده است، در نتیجه زمان پیاده‌سازی آن کاملاً قابل پیش‌بینی است. |
| **S**mall | ✅ | این داستان صرفاً بر «مشاهده» تمرکز دارد و مواردی نظیر نمره‌دهی یا ثبت غیبت در آن گنجانده نشده است. |
| **T**estable | ✅ | با بررسی تطابق لیست نمایش داده شده در سامانه با اطلاعات موجود در پایگاه داده، صحت عملکرد آن تایید می‌شود. |

---

## ۳. تعریف کلاس جدید (نقش: مدیر آموزش)
**به عنوان یک** مدیر آموزش،  
**می‌خواهم** یک درس جدید (با زمان برگزاری، ظرفیت و نام استاد) در سامانه بسازم،  
**تا** بستر لازم برای شروع فرآیند انتخاب واحد دانشجویان فراهم شود.

### بررسی INVEST
| معیار | وضعیت | توضیحات |
| :--- | :---: | :--- |
| **I**ndependent | ✅ | این قابلیت کاملاً مستقل است و مدیر آموزش می‌تواند پیش از ورود دانشجویان به سامانه، دروس را تعریف کند. |
| **N**egotiable | ✅ | ساختار و فیلدهای تعریف درس قابل گسترش است؛ مثلاً می‌توان بعداً فیلد «شماره کلاس فیزیکی» را به آن اضافه کرد. |
| **V**aluable | ✅ | این بخش نقطه شروع فرآیند ثبت‌نام است؛ بدون تعریف درس، امکان انتخاب واحد برای دانشجویان وجود نخواهد داشت. |
| **E**stimable | ✅ | طراحی یک فرم برای دریافت و ثبت اطلاعات درس، فرآیندی استاندارد است که زمان انجام آن به دقت قابل ارزیابی است. |
| **S**mall | ✅ | این داستان فقط شامل «ایجاد» درس جدید است و عملیات ویرایش یا حذف دروس به داستان‌های دیگری موکول شده است. |
| **T**estable | ✅ | با ثبت یک درس تستی در فرم و بررسی اضافه شدن آن به لیست دروس ارائه شده، عملکرد این بخش تایید می‌شود. |
