# داستان‌های کاربر - سامانه ثبت‌نام دانشگاه

## ۱. انتخاب واحد (نقش: دانشجو)
**به عنوان یک** دانشجو،  
**می‌خواهم** درس‌های دلخواهم را از لیست ترم جدید انتخاب کنم،  
**تا بتوانم** برنامه هفتگی‌ام را ببندم و خیالم از بابت کلاس‌ها راحت شود.

### بررسی با عینک INVEST
| معیار | وضعیت | چرا؟ (به زبان ساده) |
| :--- | :---: | :--- |
| **I**ndependent | ✅ | به بقیه بخش‌ها گیر نمی‌دهد! یعنی بدون اینکه منتظر سیستم پرداخت شهریه باشیم، می‌توانیم این بخش را کدنویسی کنیم. |
| **N**egotiable | ✅ | وحی منزل نیست. می‌توانیم بعداً با هم توافق کنیم که دکمه «ثبت درس» سبز باشد یا آبی، یا اینکه سقف انتخاب واحد چقدر باشد. |
| **V**aluable | ✅ | حیاتی‌ترین بخش برای دانشجوست. بدون این، کل سامانه به درد دانشجو نمی‌خورد! |
| **E**stimable | ✅ | کار عجیبی قرار نیست بکنیم؛ اضافه کردن یک اسم به یک لیست است، پس راحت می‌توانیم حدس بزنیم چقدر وقت برنامه‌نویس را می‌گیرد. |
| **S**mall | ✅ | لقمه را به اندازه برداشتیم. اینجا فقط درباره «گرفتن» درس حرف می‌زنیم، «حذف کردن» داستان دیگری است. |
| **T**estable | ✅ | تستش مثل آب خوردن است: روی دکمه کلیک می‌کنیم، اگر درس رفت توی لیست دانشجو یعنی کار می‌کند! |

## ۲. دیدن لیست کلاس (نقش: استاد)
**به عنوان یک** استاد،  
**می‌خواهم** لیست دانشجوهایی که در کلاسم ثبت‌نام کرده‌اند را ببینم،  
**تا بتوانم** برای حضور و غیاب و تعداد برگه‌های امتحانی برنامه‌ریزی کنم.

### بررسی با عینک INVEST
| معیار | وضعیت | چرا؟ (به زبان ساده) |
| :--- | :---: | :--- |
| **I**ndependent | ✅ | این فقط یک صفحه نمایش است و کاری به سیستم پیچیده ثبت‌نام دانشجوها ندارد. |
| **N**egotiable | ✅ | منعطف است. مثلاً استادها ممکن است بعداً درخواست کنند که عکس دانشجوها هم کنار اسمشان نمایش داده شود. |
| **V**aluable | ✅ | استاد را از پرسیدن مداومِ «شما دانشجوی این کلاسی؟» در جلسات اول نجات می‌دهد. |
| **E**stimable | ✅ | یک جستجوی ساده در پایگاه داده است تا اسامی را بیاورد، پس زمان انجامش کاملاً مشخص است. |
| **S**mall | ✅ | فقط برای «دیدن» است. نمره‌دهی یا غیبت زدن، کارهای بزرگتری هستند که اینجا جاشون نیست. |
| **T**estable | ✅ | لیست روی صفحه را با لیست واقعی دیتابیس چک می‌کنیم؛ اگر کسی جا نیفتاده بود، تست پاس می‌شود. |

## ۳. تعریف کلاس جدید (نقش: مدیر آموزش)
**به عنوان یک** مدیر آموزش،  
**می‌خواهم** یک درس جدید (با ساعت، ظرفیت و اسم استاد) در سامانه بسازم،  
**تا** اصلاً درسی وجود داشته باشد که دانشجوها بتوانند آن را انتخاب کنند!

### بررسی با عینک INVEST
| معیار | وضعیت | چرا؟ (به زبان ساده) |
| :--- | :---: | :--- |
| **I**ndependent | ✅ | مدیر آموزش می‌تواند وسط تابستان که هیچ دانشجویی در سایت نیست، این کلاس‌ها را تعریف کند. |
| **N**egotiable | ✅ | اگر فردا روزی خواستیم مشخص کنیم که فلان درس در «ساختمان مهندسی» برگزار می‌شود، راه برای اضافه کردنش باز است. |
| **V**aluable | ✅ | نقطه شروع ماجراست. بدون تعریف درس، انتخاب واحدی در کار نخواهد بود. |
| **E**stimable | ✅ | ساختن یک فرم ساده برای گرفتن اطلاعات درس است که زمان ساختش کاملاً قابل تخمین است. |
| **S**mall | ✅ | فقط یک کار انجام می‌دهد: ساخت درس جدید. ویرایش یا حذف درس را قاطی این کار نکردیم. |
| **T**estable | ✅ | فرم را پر می‌کنیم؛ اگر درس در لیست ترم جدید اضافه شد، یعنی همه‌چیز درست کار می‌کند. |
